---
title: omp-squad
description: A control plane for a fleet of Oh My Pi coding agents — one per git worktree — driven from a terminal TUI and a web dashboard.
---

**omp-squad** runs many [Oh My Pi](https://omp.sh) (`omp`) coding agents in parallel and gives you
one place to watch and steer them. Each agent works in its own `git worktree` as a real
`omp --mode rpc` child process; a single `SquadManager` derives every agent's live status from its
event stream and broadcasts it to two interchangeable surfaces — a terminal **TUI** and a **web
dashboard**.

The bottleneck of N parallel agents isn't the agents — it's *your attention*. omp-squad is built to
route it: see at a glance who is working, who is idle, and **who is blocked waiting for you**, then
drop into any agent and steer it.

```text
        omp-squad (one process)
        ┌─────────────────────────────────────────────┐
        │  SquadManager  ── roster, status, transcript │
        │     │                                        │
        │     ├── RpcAgent ──▶ omp --mode rpc  (wt #1) │   each agent =
        │     ├── RpcAgent ──▶ omp --mode rpc  (wt #2) │   its own worktree
        │     └── RpcAgent ──▶ omp --mode rpc  (wt #3) │   + RPC child process
        │     │                                        │
        │     ├── SquadServer (HTTP + WS) ──▶ browser  │
        │     └── SquadTui    (terminal)               │
        └─────────────────────────────────────────────┘
```
