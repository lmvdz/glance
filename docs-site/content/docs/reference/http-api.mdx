`src/server.ts` (`SquadServer.start`) serves one Bun server: a static SPA shell, a token-gated REST surface under `/api`, and a `/ws` WebSocket carrying the live `SquadEvent` stream. Every shape below is traced to the `fetch`/`websocket` handlers.

## Authentication

**File mode** (no `DATABASE_URL`): every `/api/*` request and the `/ws` handshake must present the daemon's bearer token, either as `Authorization: Bearer <token>` or a `?token=<token>` query parameter. Missing or wrong ⇒ `401 unauthorized`. **DB mode**: httpOnly cookie sessions issued by BetterAuth replace the token (`/api/auth/*` is delegated to the BetterAuth handler), and all non-GET requests also pass an Origin check.

Either way, the resolved identity maps to a **role tier** — `viewer ⊂ operator ⊂ admin` (`src/authz.ts`). The default rule is *GET → viewer, non-GET → operator*; exceptions are flagged per-route below (`/api/upgrade`, feature-flag writes, capability mutations, land/vision are **admin**).

The bootstrap surface loads **without** auth so the PWA can install before sign-in:

- `GET /` and `GET /index.html` — the SPA shell (the built React `webapp/dist` when `GLANCE_WEBAPP=1`, else the zero-build fallback), plus `/assets/*` for the built bundle.
- `GET /manifest.webmanifest`, `/sw.js`, `/icon-192.png`, `/icon-512.png`, `/icon-maskable-512.png` — the public PWA assets.
- `GET /api/auth/mode` — pre-login probe: file vs DB mode, `allowSignup`, configured social providers, SSO.
- `GET /llms.txt` and `GET /openapi.json` — machine-readable capability-API discovery.
- `GET /feedback/widget.js` and `POST /api/feedback/items` — the public feedback intake (only when `GLANCE_FEEDBACK=1`; rate-limited per campaign + IP).
- `POST /api/workos/webhook` — the WorkOS Directory Sync (SCIM) webhook, HMAC-verified with `WORKOS_WEBHOOK_SECRET` instead of a session.

Any other unmatched path returns `404 not found`.

## Per-agent

`:id` is URL-encoded. Routes that resolve an agent return `404 no such agent` when it is missing.

| Method | Path | Purpose | Response |
| --- | --- | --- | --- |
| GET | `/api/agents/:id/transcript` | Full transcript. | [`TranscriptEntry[]`](/docs/reference/types) |
| GET | `/api/agents/:id/subagents` | Spawned fan-out children. | subagent list |
| GET | `/api/agents/:id/receipts` | Per-run receipts. | `RunReceipt[]` |
| GET | `/api/agents/:id/commands` | Slash commands available to the agent. | `CommandInfo[]` (or `[]`) |
| GET | `/api/agents/:id/diff` | Unlanded worktree diff. | diff payload |
| GET | `/api/agents/:id/tree` | Worktree file tree. | tree payload |
| POST | `/api/agents/:id/land` | Commit + merge the branch to main. | land result; see below |
| POST | `/api/agents/:id/verify` | Run the repo's detected acceptance command as a land-proof. | proof result; `422` if none detected |
| POST | `/api/agents/:id/vision` | Capture screenshot evidence (never gates a land). **Admin-tier.** | `{ artifacts }`; `422` if no URL; `400` if the URL fails the SSRF guard |

**`/land`** accepts an optional JSON body `{ message?, force? }`. Unless `force` is `true`, a proof gate runs first; if it fails the route returns `409` with `{ ok: false, committed: false, merged: false, message, detail }`. Work-in-progress is committed only when the agent is not busy (`working`/`starting`/`input`). On a successful land the agent's tracking Plane issue is closed (best-effort, idempotent).

**`/vision`** takes `{ url? }`; absent that it falls back to `$GLANCE_APP_URL`. With neither it returns `422`. It is **admin-tier** — it drives the daemon's headless browser, which reaches the daemon's network position. To stop that becoming an SSRF vector, the target is validated: only `http(s)` is allowed, and the host is DNS-resolved and **rejected with `400`** if it lands on a private / loopback / link-local / reserved range (RFC1918, `127.0.0.0/8`, `169.254.0.0/16` incl. the cloud-metadata IP, `::1`, `fc00::/7`, `fe80::/10`, …). The one exception is the origin of `$GLANCE_APP_URL`, which is allowlisted — set it to screenshot a trusted local app (e.g. `http://localhost:3000`).

## Presence, leases & federation

| Method | Path | Purpose | Response |
| --- | --- | --- | --- |
| GET | `/api/presence?repo=` | Presence for one repo, or all active agents. | presence entries |
| GET | `/api/leases?repo=` | File leases for a repo (defaults to the daemon cwd). | `LeaseEntry[]` |
| GET | `/api/fabric?repo=` | Agent fabric view for a repo including live leases. | fabric payload |
| GET | `/api/federation` | Roster-of-rosters: self + peers + branch collisions. | `FederationSnapshot` |
| POST | `/api/federation/command` | Send an addressed remote command to a peer operator's daemon (`{ to, cmd }`); the receiver re-authorizes via whois-verified actor + RBAC. Audited as `federation.command`. | `{ ok }`; `403` on RBAC denial |

**`/api/leases`** returns `LeaseEntry[]` — soft advisory file-edit claims keyed by `(repo, file, session)`. Each entry carries `{ id, repo, file, operator, session, host, since, heartbeat }`. In file mode, the full machine-wide registry is returned for the requested repo (or `cwd`). In DB-registry mode the result is scoped to repos the calling org's fleet actually works on — another org's leases are never visible.

**`/api/fabric`** returns the agent fabric view for the given repo, including live file leases. Like leases, the scope is always the calling org's agents — never cross-org.

**`/api/federation`** returns a `FederationSnapshot`:

```ts
interface FederationSnapshot {
  /** ws:// coordinator URL this daemon is connected to, or null when federation is off or in DB mode. */
  coordinator: string | null;
  /** self (pinned head) + live peers, deduped by operator id. */
  operators: OperatorPresence[];
  /** Repos where agents owned by DIFFERENT operators share a branch — a collision to resolve. */
  collisions: Collision[];
}
```

In file mode, `operators` includes the local daemon's roster plus any live peers gossiped from the coordinator. In DB-registry mode, `operators` contains only the calling org's own roster (no cross-host coordinator feed per org) and `coordinator` is always `null`. Previously this endpoint returned empty data in DB mode; it now always returns the org's own agents.

## Automation (background loops)

| Method | Path | Purpose | Response |
| --- | --- | --- | --- |
| GET | `/api/automation` | Background-loop observability: recent events + per-loop rollups. | `{ events: AutomationEvent[], rollup: AutomationRollupRow[] }` |

Query parameters:

| Parameter | Type | Default | Description |
| --- | --- | --- | --- |
| `loop` | `"scout" \| "observer" \| "opportunity" \| "dispatch"` | all loops | Filter to one named background loop. |
| `windowMs` | number | `3600000` (1 h) | Size of the rollup window in milliseconds. |
| `meaningful` | `"1"` | off | When `1`, drop heartbeat events (ticks with no work done) from the event feed. |
| `limit` | number | `200` | Cap on the number of `events` returned (newest first). |
| `sinceMs` | number | unset | Only return events newer than `now - sinceMs`. |

`events` is a list of `AutomationEvent` (see [Types](/docs/reference/types)), newest first. `rollup` is a per-loop aggregate row over the requested window — it always covers all four loops (rows for silent loops have zero counts).

## Commands

`POST /api/command` is the single mutation endpoint the CLI and surfaces post to. The body is a [`ClientCommand`](/docs/reference/types).

| `cmd.type` | Result |
| --- | --- |
| `create` | Creates the agent with `track: true`; returns the new [`AgentDTO`](/docs/reference/types). |
| `commission` | Runs the commissioning loop with `{ install: true }`; returns a `CommissionResult`. |
| anything else | `manager.applyCommand(cmd)`; returns `{ ok: true }`. |

Malformed JSON returns `400 bad json`.

## WebSocket protocol

Connect to `/ws` (token-gated). The protocol is JSON frames in both directions, typed by [`SquadEvent`](/docs/reference/types) (server→client) and [`ClientCommand`](/docs/reference/types) (client→server).

### On connect

The server immediately sends the initial snapshot:

```ts
// 1. a full roster snapshot, stamped with the served-UI version
{ type: "roster", agents: AgentDTO[], version: string }
// 2. then one per agent that has slash commands available
{ type: "commands", id: string, commands: CommandInfo[] }
```

### Server → client

Every `SquadEvent` is broadcast to all connected sockets; `roster` events are re-stamped with the live `uiVersion` so tabs can self-refresh after an upgrade:

```ts
type SquadEvent =
	| { type: "roster"; agents: AgentDTO[]; version: string }
	| { type: "agent"; agent: AgentDTO }
	| { type: "removed"; id: string }
	| { type: "transcript"; id: string; entry: TranscriptEntry }
	| { type: "log"; level: "info" | "warn" | "error"; text: string }
	| { type: "commands"; id: string; commands: CommandInfo[] }
	| { type: "features-changed" }
	| { type: "comment"; comment: ArtifactCommentDTO }
	| { type: "comment-resolved"; id: string; resolvedAt: number }
	| { type: "audit"; entry: AuditEntry }
	| { type: "automation"; event: AutomationEvent };
```

### Client → server

Send a `ClientCommand`. `subscribe` is special: it triggers a **unicast** replay of that agent's transcript (one `transcript` event per entry) to the requesting socket only. Every other command is applied via `manager.applyCommand`. Unparseable frames are silently dropped.

```ts
type ClientCommand =
	| { type: "prompt"; id: string; message: string }
	| { type: "answer"; id: string; requestId: string; value: string }
	| { type: "interrupt"; id: string }
	| { type: "kill"; id: string }
	| { type: "restart"; id: string }
	| { type: "remove"; id: string; deleteWorktree?: boolean }
	| { type: "create"; options: CreateAgentOptions }
	| { type: "snapshot" }
	| { type: "subscribe"; id: string }
	| { type: "commission"; spec: CommissionSpec };
```

See [Types](/docs/reference/types) for the field-level definitions of every payload above.
