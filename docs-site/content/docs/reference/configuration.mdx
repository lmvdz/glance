omp-squad takes no config file — everything is environment variables plus a small on-disk state directory. Every variable and default below is traced to its read site in `src/`. Variables with no default are simply unset (the feature stays off or falls back as noted).

## Daemon

Read by `src/index.ts` (`cmdUp`) and `src/server.ts` when starting the daemon.

<TypeTable
	type={{
		'OMP_SQUAD_PORT': { type: 'number', default: '7878', description: 'Port the daemon binds. Backs the --port flag.' },
		'OMP_SQUAD_HOST': { type: 'string', default: '127.0.0.1', description: 'Bind address. Set 0.0.0.0 to reach the dashboard across your LAN. Backs the --host flag.' },
		'OMP_SQUAD_TLS_CERT': { type: 'path', default: 'unset', description: 'PEM cert path. TLS is enabled in-process only when both CERT and KEY are set; the scheme becomes https.' },
		'OMP_SQUAD_TLS_KEY': { type: 'path', default: 'unset', description: 'PEM key path. Required together with OMP_SQUAD_TLS_CERT.' },
		'OMP_SQUAD_STATE_DIR': { type: 'path', default: '~/.omp/squad', description: 'Directory for state.json, the access token, and push keys.' },
		'OMP_SQUAD_AUTO_SUPERVISE': { type: 'flag', default: 'on', description: 'The standing auto-supervisor loop. On unless set to 0 (or the --no-supervise flag is passed).' },
		'OMP_SQUAD_SUPERVISE_MODEL': { type: 'string', default: 'omp default', description: 'Model the auto-supervisor answerer runs as. Unset uses the omp default.' },
	}}
/>

<Callout type="warn">
`OMP_SQUAD_AUTO_SUPERVISE` (with the underscore) gates the standing supervisor process started by `up`. It is **not** the same variable as `OMP_SQUAD_AUTOSUPERVISE` (no underscore) below, which controls the manager's per-request deterministic auto-answer. They are independent switches.
</Callout>

## Resource limits & watchdog

<TypeTable
	type={{
		'OMP_SQUAD_RESOURCE_GATE': { type: 'flag', default: 'off', description: 'Enable load-aware admission backoff. Off by default — the WIP cap alone is enough for most setups.' },
		'OMP_SQUAD_MAX_LOAD_PER_CPU': { type: 'number', default: '1.5 (admission gate) / 2.0 (watchdog)', description: 'load1 / logical-CPU threshold. 		'OMP_SQUAD_MAX_RSS_MB': { type: 'number', default: '1024', description: 'Daemon RSS ceiling in MB for the watchdog. Crossing it emits a log warning and is surfaced in /api/health warnings. Not a hard kill limit.' },
		'OMP_SQUAD_MIN_FREE_RATIO': { type: 'number', default: '0.1', description: 'Minimum free-memory fraction (0–1) before the host is considered pressured. Shared by the admission gate and the watchdog.' },

## Payments

Reward disbursement for the Feedback Loop feature. Read by `src/payments/tremendous.ts` and `src/payments/index.ts`. If `OMP_SQUAD_TREMENDOUS_API_KEY` is unset, the daemon falls back to a Manual recorder (no network; the operator records out-of-band payouts).

<TypeTable
	type={{
		'OMP_SQUAD_TREMENDOUS_API_KEY': { type: 'string', default: 'unset', description: 'Tremendous API Bearer token. Its presence is the activation switch — set it to enable real reward disbursement via the Tremendous Orders API. Unset ⇒ ManualProvider (no network, operator records payouts by hand).' },
		'OMP_SQUAD_TREMENDOUS_FUNDING_SOURCE_ID': { type: 'string', default: 'unset', description: 'Tremendous funding source (balance / account) that funds each payout. Required for any real disbursement; missing ⇒ every payout resolves as failed with a clear error.' },
		'OMP_SQUAD_TREMENDOUS_CAMPAIGN_ID': { type: 'string', default: 'unset', description: 'Tremendous campaign id that controls which redemption options (gift cards, PayPal, ACH, Visa, …) the reward recipient sees. Required for real disbursement.' },
		'OMP_SQUAD_TREMENDOUS_ENV': { type: '"sandbox" | "production"', default: '"sandbox"', description: 'Selects the Tremendous API base URL. sandbox → testflight.tremendous.com (default, safe for testing); production → api.tremendous.com (live money).
