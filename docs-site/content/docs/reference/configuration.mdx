omp-squad takes no config file — everything is environment variables plus a small on-disk state directory. Every variable and default below is traced to its read site in `src/`. Variables with no default are simply unset (the feature stays off or falls back as noted).

## Daemon

Read by `src/index.ts` (`cmdUp`) and `src/server.ts` when starting the daemon.

<TypeTable
	type={{
		'OMP_SQUAD_PORT': { type: 'number', default: '7878', description: 'Port the daemon binds. Backs the --port flag.' },
		'OMP_SQUAD_HOST': { type: 'string', default: '127.0.0.1', description: 'Bind address. Set 0.0.0.0 to reach the dashboard across your LAN. Backs the --host flag.' },
		'OMP_SQUAD_TLS_CERT': { type: 'path', default: 'unset', description: 'PEM cert path. TLS is enabled in-process only when both CERT and KEY are set; the scheme becomes https.' },
		'OMP_SQUAD_TLS_KEY': { type: 'path', default: 'unset', description: 'PEM key path. Required together with OMP_SQUAD_TLS_CERT.' },
		'OMP_SQUAD_STATE_DIR': { type: 'path', default: '~/.omp/squad', description: 'Directory for state.json, the access token, and push keys.' },
		'OMP_SQUAD_AUTO_SUPERVISE': { type: 'flag', default: 'on', description: 'The standing auto-supervisor loop. On unless set to 0 (or the --no-supervise flag is passed).' },
		'OMP_SQUAD_SUPERVISE_MODEL': { type: 'string', default: 'omp default', description: 'Model the auto-supervisor answerer runs as. Unset uses the omp default.' },
	}}
/>

<Callout type="warn">
`OMP_SQUAD_AUTO_SUPERVISE` (with the underscore) gates the standing supervisor process started by `up`. It is **not** the same variable as `OMP_SQUAD_AUTOSUPERVISE` (no underscore) below, which controls the manager's per-request deterministic auto-answer. They are independent switches.
</Callout>

## Autonomy

Drive how hands-free the fleet runs — the orchestrator, the WIP cap, repair/resolve budgets, and the manager-side auto-answer.

<TypeTable
	type={{
		'OMP_SQUAD_AUTODRIVE': { type: 'flag', default: 'off', description: 'Enables the Orchestrator autonomy loop (src/orchestrator.ts). On when set to any value.' },
		'OMP_SQUAD_MAX_WIP': { type: 'number', default: '6', description: 'Ceiling on simultaneously live agents (src/scheduler.ts). Spawns past it are refused unless queued.' },
		'OMP_SQUAD_REPAIR_BUDGET': { type: 'number', default: '3', description: 'Retry attempts the resolver allows before escalating a failed gate to a human (src/resolver.ts).' },
		'OMP_SQUAD_AUTORESOLVE': { type: 'flag', default: 'off', description: 'Auto-resolve rebase conflicts during land instead of escalating (src/land.ts). On when set to any value.' },
		'OMP_SQUAD_QUEUE_ON_FULL': { type: 'flag', default: 'off', description: 'When the WIP cap is hit, queue the spawn (returning a synthetic queued DTO) instead of throwing (src/squad-manager.ts).' },
		'OMP_SQUAD_AUTOSUPERVISE': { type: 'flag', default: 'off', description: 'Manager-side deterministic auto-answer of safe pending requests (src/squad-manager.ts). On when set to any value.' },
		'OMP_SQUAD_AUTOSUPERVISE_BUDGET': { type: 'number', default: '5', description: 'Per-agent cap on those auto-answers before a request is left for a human.' },
	}}

## Plane

Read by `src/plane.ts`; if neither an API key nor a workspace is set, Plane integration is disabled (issue routes return `501`). Alternate secret names are accepted so the daemon can read a pasted `plane.env` directly.

<TypeTable
	type={{
		'PLANE_API_KEY': { type: 'string', default: 'unset', description: 'API key. PLANE_API_TOKEN is accepted as an alias.' },
		'PLANE_WORKSPACE': { type: 'string', default: 'unset', description: 'Workspace slug. PLANE_WORKSPACE_SLUG is accepted as an alias.' },
		'PLANE_BASE_URL': { type: 'url', default: 'https://api.plane.so', description: 'API base. A trailing /api/v1 is stripped since every call appends it.' },
		'PLANE_PROJECT_ID': { type: 'string', default: 'unset', description: 'Fallback project id when a repo is not in PLANE_PROJECT_MAP.' },
		'PLANE_PROJECT_MAP': { type: 'json', default: 'unset', description: 'JSON object mapping repo path → Plane project id. Invalid JSON is ignored (treated as empty).' },
		'PLANE_APP_URL': { type: 'url', default: 'derived', description: 'Web app base for deep links. When unset it is derived from PLANE_BASE_URL (app.plane.so for the hosted API).' },
	}}
/>

See [Plane integration](/docs/integrations/plane).
