The `glance` binary is a thin client over a running daemon (`omp-squad` is installed as a legacy alias of the same entry point). `up` starts that daemon; the daemon-facing commands reach it over HTTP at `http://127.0.0.1:<port>`, sending the bearer token read from `<state-dir>/access-token` (see [Configuration](/docs/reference/configuration)). If no daemon is listening they print `No squad daemon on <base>. Start one with: glance up` and exit non-zero. `open`, `who`, `plan-validate`, `curate-plane`, and `help` work without a daemon.

## Argument grammar

`src/index.ts` parses `argv` with one rule (`parseArgs`):

- `--key value` — a flag whose next token does not start with `--` takes that token as its string value.
- `--key` — a flag followed by another `--flag` (or end of args) becomes the boolean `true`.
- anything not starting with `--` is a positional, in order.

So `--limit 10` sets `limit="10"` while a bare `--no-tui` sets `no-tui=true`. Values are always strings; numeric flags (`--port`, `--limit`) are coerced with `Number()`.

<Callout type="info">
Running `glance` with no command is identical to `glance up` — the daemon is the default.
</Callout>

## `up`

```bash
glance up [--port N] [--no-tui] [--restore] [--host <addr>] [--no-supervise]
```

Starts a `SquadManager`, a `SquadServer` (HTTP + WebSocket), the external-session tracker, and — unless opted out — the standing auto-supervisor. Prints the dashboard URL and access token; on a TTY it then runs the terminal TUI, otherwise it stays headless and prints every reachable URL with a `?token=` query so you can sign in from another device.

| Flag | Default | Description |
| --- | --- | --- |
| `--port N` | `7878` (or `$GLANCE_PORT`) | Port the daemon binds. |
| `--no-tui` | TUI on if `stdin` is a TTY | Run headless; never launch the TUI. Headless is also forced automatically when `stdin` is not a TTY. |
| `--restore` | off | Reattach to persisted agents from `state.json`; prints `restored N agent(s)`. |
| `--host <addr>` | `127.0.0.1` (or `$GLANCE_HOST`) | Bind address. Set `0.0.0.0` to reach it across your LAN. |
| `--no-supervise` | supervisor on | Don't start the auto-supervisor loop. Equivalent to `GLANCE_AUTO_SUPERVISE=0`. |

TLS is enabled in-process only when both `$GLANCE_TLS_CERT` and `$GLANCE_TLS_KEY` are set; the scheme then becomes `https`. Federation joins a coordinator when `$GLANCE_COORDINATOR` is set. See [Configuration](/docs/reference/configuration) for all of these.

<Callout type="warn">
A bearer token is auto-generated in the state dir on first boot and printed on every start. It guards `/ws` and everything under `/api`. The static SPA shell loads without it so the PWA can install before sign-in.
</Callout>

## `add`

```bash
glance add [<repo>] [flags]
```

Spawns one agent. The `<repo>` positional defaults to the current working directory (`process.cwd()`). Before spawning, the CLI checks presence and warns on stderr if any agent (squad or raw omp session) is already active on that repo. On success it prints the new agent's name, status, id, and worktree path. The flags below map onto `CreateAgentOptions` and are sent as a `create` command; the defaults shown are applied by the daemon (`SquadManager.create`), not the CLI.

| Flag | Default | Description |
| --- | --- | --- |
| `--name <s>` | `agent-N` (incrementing) | Agent name. |
| `--branch <s>` | `squad/<name>` | Worktree branch to create/checkout. |
| `--model <s>` | omp default | Model, fuzzy-matched (e.g. `opus`, `gpt-5.2`). |
| `--approval <m>` | `write` | One of `always-ask`, `write`, `yolo`. |
| `--thinking <l>` | `low` | One of `minimal`, `low`, `medium`, `high`, `xhigh`. |
| `--task <s>` | none | Instruction sent once the agent is ready. |
| `--workflow <name\|path>` | none | Run a bundled workflow (`research-plan-implement`, `plan-implement`, `fan-out`) or a `.fabro` path; `--task` becomes the goal. |
| `--verify <cmd>` | none | Wrap `--task` in an implement → verify → fixup loop; the gate passes on exit code `0`. |
| `--sandbox <image>` | none | Run the agent inside a container from `<image>` (mounts the worktree). |
| `--acp` | off | Run an ACP runtime (`auggie --acp` by default) instead of `omp --mode rpc`. `--runtime acp` is equivalent. |
| `--plain` | off | Skip auto-routing; spawn a plain agent with no verify/plan/fan-out inference. |

<Callout type="info">
With a `--task` and no `--plain`/`--workflow`/`--verify`/`--sandbox`, the daemon auto-routes the task — it may pick a workflow, a verify loop, or a thinking level for you. `--plain` (which sets `autoRoute=false`) turns that off.
</Callout>

## `list` (`ls`)

```bash
glance list
```

Fetches `GET /api/agents` and prints an aligned table: `status`, `name`, `branch` (or `—`), a `⛔N` pending-count badge, and the current `activity` (falling back to the active todo). Prints `no agents` when the roster is empty.

## `prompt` (`say`)

```bash
glance prompt <id> <message...>
```

Sends a `prompt` command to agent `<id>`; all positionals after the id are joined with spaces into the message. Both `<id>` and a non-empty message are required (else it prints `usage:` and exits `1`). Prints `sent` on success.

## `kill` (`stop`)

```bash
glance kill <id>
```

Sends a `kill` command to terminate the agent's process without removing it from the roster. The agent entry remains visible with status `stopped`. Use `rm` to also drop it from the roster. Prints `killed` on success.

## `rm` (`remove`)

```bash
glance rm <id> [--delete-worktree]
```

Sends a `remove` command. `--delete-worktree` (boolean) also tears down the agent's git worktree; omit it to keep the worktree on disk. Prints `removed` on success.

## `who`

```bash
glance who [repo]
```

Reads the shared presence registry. With a `repo` it lists agents working that repo; with no argument it lists all active agents (squad members **and** raw omp sessions). Each row shows source, `operator/agent`, repo name, optional branch, and a humanized "N ago" since the last heartbeat. Prints `nobody is working on <repo>` / `no active agents` when empty.

## `logs`

```bash
glance logs <id> [--limit N]
```

Fetches `GET /api/agents/<id>/transcript` and prints the last `N` entries (kind in upper case, then text). `<id>` is required.

| Flag | Default | Description |
| --- | --- | --- |
| `--limit N` | `40` | How many trailing transcript entries to print. |

## `automation` (`auto`)

```bash
glance automation [--window <dur>] [--loop <name>] [--limit N] [--json]
```

Fetches `GET /api/automation` and prints a summary of what the daemon's background loops (Scout, Observer, Opportunity, Dispatcher) have been doing — the terminal twin of the automation panel in the dashboard.

| Flag | Default | Description |
| --- | --- | --- |
| `--window <dur>` | `1h` | Rollup window. Accepted values: `15m`, `1h`, `6h`, `24h`. |
| `--loop <name>` | all loops | Filter output to one loop: `scout`, `observer`, `opportunity`, or `dispatch`. |
| `--limit N` | `20` | Maximum events to show in the recent feed. |
| `--json` | off | Emit raw JSON (`{ events, rollup }`) instead of the formatted table. |

Output: a per-loop summary table (events, LLM calls, issues filed, candidates found, last activity) followed by a recent-events feed. An empty table means no background activity has been recorded yet — loops start once agents and Plane repos are configured.

## `curate-plane` (`plane-curator`)

```bash
glance curate-plane [repo] [--file]
```

Reads Plane issues for the given repo (defaults to `cwd`) and groups recurring or related issues into unified fix proposals — it talks to Plane directly, no daemon needed. Prints `Plane is not configured or unreachable` and exits 1 if Plane is unavailable.

| Flag | Default | Description |
| --- | --- | --- |
| `--file` | off | Also **file one `[curator]` do-not-auto-land Plane issue per cluster**, so the fleet fixes one unified issue instead of dispatching duplicates. |

## `plan-validate` (`validate-plan`)

```bash
glance plan-validate <dir> [--json]
```

Offline validator for a `plans/<name>/` directory: reads the concern docs off disk and reports dependency-graph **cycles** and **dangling deps** — no daemon, no network. Clean → a `✓` line and exit `0`; issues → one `[kind] message` line each and exit `1`.

| Flag | Default | Description |
| --- | --- | --- |
| `--json` | off | Emit `{ dir, issues }` JSON instead of lines (still exits `1` when issues exist). |

## `open`

```bash
glance open
```

Prints the daemon base URL `http://127.0.0.1:<port>` (honoring `--port`) and nothing else — no token, no path. Useful for piping into a browser launcher.

## `commission` (`hire`)

```bash
glance commission <name> --purpose <s> [flags]
```

Authors and validates a Flue worker, then onboards it as a `flue-service` member if it passes the acceptance gate. Both the `<name>` positional and `--purpose` are required (else it prints `usage:` and exits `1`). It prints each gate check (`✓` / `✗` / `·`) and, on success, the onboarded member's name (with `verified` when the acceptance gate ran), id, and directory. A failed gate prints `rejected — gate failed; worker left at <dir>` and exits `1`.

| Flag | Default | Description |
| --- | --- | --- |
| `--purpose <s>` | required | What the worker does. |
| `--model <spec>` | `false` | Model specifier, or the literal `false` for a deterministic (no-LLM) worker. |
| `--target <t>` | `node` | Deploy target: `node` or `cloudflare` (anything else falls back to `node`). |
| `--capabilities <a,b>` | none | Comma-separated least-privilege tool allowlist, recorded in the manifest. |
| `--accept-payload <json>` | none | Acceptance input (parsed as JSON). |
| `--accept-expect <json>` | none | Expected-result subset (parsed JSON object); only honored alongside `--accept-payload`. |

See [Commissioning](/docs/commissioning) for the full authoring loop.

## `help`

```bash
glance help        # also: -h, --help
```

Prints the usage banner.
