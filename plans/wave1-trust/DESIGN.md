# Design: wave1-trust

_Origin: plans/research-direct-vs-glance/BRIEF.md Wave 1 ("Truth and finishing", PR #28). Adversarial design 2026-07-03: sonnet designer → 2× fable red team (26 findings, 9 critical) → fable arbiter. Landscape, draft, and both red-team critiques archived in the session job dir; every contested claim below was re-verified against source by the arbiter._

## Approach

**Mode dispatch lives below the manager's control machinery, above the merge primitive.** `landBranch` (squad-manager.ts:1742-44) becomes the universal mode-dispatching seam, and `landFeature` (:1608) is rerouted through it — closing the two-worlds-inside-the-daemon hole (feature lands would otherwise keep local-merging while single agents push PRs). Because dispatch sits *below* `manager.land()` (:1631-95), PR mode automatically inherits proofGate, force-requires-reason, fail-cap, forced-land audit, landReady clearing, and closeLandedIssue — nothing is re-implemented. In PR mode, landReady (markLandReady :1717 / staged :1669) additionally floats an idempotent push + `gh pr create --draft`; the Land button drives a new `landAgentPr` primitive that is **synchronous end-to-end**: ensure-PR → proofGate re-check against branch tip → fetch → **scratch-merge gate** (acceptance + regression gate run on a disposable detached worktree of origin/&lt;default&gt; merged with the branch — scoped at merge-click only) → `gh pr ready` + `gh pr merge` → fetch + per-method reachability assertion → DoneProof + close + cleanup, all inside `withRepoLandLock`. Since `gh pr merge` is synchronous, truth is written at the moment of merge; the async reconciler is demoted to a backstop for out-of-band merges (the user clicking merge in the GitHub UI — the actual world-split case), driven from a **durable pending-PR ledger written at push time**, homed in a small always-on manager loop, not the toggleable Observer.

**Done-truth is a ledger, not arithmetic.** A new DoneProof ledger (tri-state `verified`) is the single artifact that authorizes a Done write. It is written at the manager layer (where `dto.branch` + `dto.issue` + `LandResult` coexist), consulted by `closeLandedIssue`, by the write-half of `issueAlreadyDone` (the *skip*-half stays proofless — gating the skip would re-open the PR #18 stale-re-dispatch incident), by plan-sync's `⇒done` branch, and — critically — **consulted FIRST by Observer checks 2/4, `agentHasUnlandedWork`, and the reaper**, because squash/rebase merges make rev-list arithmetic permanently wrong. All remaining arithmetic consumers move to ONE origin-aware `aheadOfBase` primitive. PR-mode worktrees fork from a freshly-fetched origin/&lt;default&gt; via a new real `startPoint` param on `addWorktree` (verified: today's `base` param is a directory, not a git ref — branches fork from local HEAD).

**Auto mode is gated on proven safety, not optimism.** `OMP_SQUAD_LAND_MODE=auto` (default) resolves to PR mode per-repo only when a five-point probe passes: owner/repo parseable from origin, `gh repo view owner/repo` OK (host-alias-proof — the flagship origin is `git@github.com-personal:...`, so URL-host checks fail exactly there), `git push --dry-run` OK, current local branch == remote default branch, and **local default is an ancestor of origin/&lt;default&gt;** — divergence itself forces local mode with a loud per-repo log. Empirical finding at design time: local main and origin/main are currently **0/0 converged** (the ~112-commit drift was healed by the recent PR cycle), so the flagship gets PR mode day one and migration reduces to a preflight guard preventing recurrence. Goals 2 and 3 are mode-independent and ship in the first batches so trust lands even where PR mode doesn't: gate-class is stamped at the *real* emitter — `raiseGate`'s `gate_` id prefix (workflow-driver.ts:295-310) detected in `onUi` — and enforced in both auto-answer engines with one shared test; the regression gate flips to default-ON and, via the scratch-merge gate, **runs in PR mode too**, so the flip is not dead code on exactly the repos the wave targets.

## Data shapes (final)

DoneProof (new src/done-proof.ts, per-repo ledger file mirroring land-ledger.ts, keyed by branch, indexed by issueIdentifier): {branch, repo (repoIdentity key), issueId?, issueIdentifier?, mode "local"|"pr", method? (pr only), commit (branch tip proven included), mergeCommit? (pr only), baseRef ("HEAD"|"origin/main"), verified "green"|"red-baseline"|"unverified", detail, provenAt, prNumber?, prUrl?}.

PendingPr (per-repo file written at push+create time): {branch, repo, prNumber, prUrl, issueId?, issueIdentifier?, agentId?, createdAt, state "open"|"merged"|"closed", mergedAt?, proofAt?, issueClosedAt?} — the last three are the idempotent reconcile keys.

LandResult (land.ts:18-43, additive): mode?, pushed?, prUrl?, prNumber?, prState? ("draft"|"open"|"merged"|"closed"). `merged:true` in PR mode ONLY after the post-merge reachability assertion.

PendingRequest (types.ts:32-43, additive): `gateClass?: boolean` — stamped in onUi when `req.id.startsWith("gate_")` OR title starts with `GATE:`.

AgentDTO (additive): prUrl?, prNumber?, prState?.

addWorktree (worktree.ts:68): new optional `startPoint?: string` appended as the git start-point when creating (`worktree add -b <branch> <dir> <startPoint>`); resolveWorktree threads it; create() passes fetched `origin/<default>` in PR mode.

Env flags (final defaults): OMP_SQUAD_LAND_MODE=auto (auto|pr|local; auto ⇒ pr only if 5-point probe passes) · OMP_SQUAD_PR_MERGE_METHOD=merge · OMP_SQUAD_PR_DRAFT=1 · OMP_SQUAD_PR_BASE="" (default = gh defaultBranchRef) · OMP_SQUAD_REGRESSION_GATE flipped to `!== "0"` (default ON) · OMP_SQUAD_GATE_SANDBOX_STRICT unchanged · OMP_SQUAD_LAND_CONFIRM unchanged (default ON).

## Risks

1. **Regression gate flip roughly doubles land latency** (two extra suite runs; PR mode adds a scratch-worktree run). Mitigation: runtime-settings escape hatch day one; scratch gate only at merge-click.
2. **Grandfathered Dones have no proof**: plan-sync writes `done (unproven)` markers for legitimately-shipped old work; issueAlreadyDone suppresses closes for terminal-without-proof docs. Mitigation: both surfaced, never silent; skip-dispatch unaffected so no re-dispatch risk; one-time operator sweep listed in the boot banner.
3. **Out-of-band GitHub-UI merge latency** (≤120s reconciler tick): stale prState badge. Mitigation: synchronous path covers all daemon-initiated merges.
4. **LLM conflict autoresolve absent in PR mode**. Mitigation: clean-automerge retry covers trailing-main; conflicts surface with exact detail; re-evaluate in Wave 2 (this is also the diagnosis' "steering lane" moment).
5. **`gh` is an environmental dependency** (rate limits, auth expiry). Mitigation: probe at boot + first land; every gh failure degrades to loud local mode or surfaced refusal, never a crash; ledger preserves state across outages.
6. **Sibling-plan merge collisions** (lifecycle-truth touches squad-manager.ts heavily). Mitigation: this wave's squad-manager edits are localized (land :1585-1744, issueAlreadyDone :694-720, onUi :3124-3148, aheadOfMain :1804-1820); no AgentStatus writes touched; coordinate land order.
